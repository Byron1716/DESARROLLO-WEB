Semántica y significado

Las etiquetas semánticas (como "header", "article", "footer") indican el rol del contenido, mejorando la accesibilidad y el SEO. entender la página.




La semántica en HTML se refiere a usar etiquetas que describen el significado y propósito del contenido (como article, nav, footer), no solo su apariencia, para que navegadores, buscadores y tecnologías de asistencia entiendan la estructura y jerarquía del sitio, mejorando la accesibilidad, SEO y mantenibilidad. El significado es la función lógica que cada elemento cumple (un párrafo es p, un encabezado es h1, etc.), a diferencia de etiquetas genéricas como div que no dan contexto.

Significado y Semántica en HTML

Semántica:
Es el estudio del significado de las palabras y, en HTML, el significado inherente de las etiquetas y su relación lógica.
Propósito: Usar etiquetas que describan qué es el contenido, no cómo se ve.

Ejemplos de Etiquetas Semánticas
h1 - h6: Títulos y subtítulos.
p: Párrafos de texto.
a: Enlaces.
img: Imágenes.
header: Encabezado de una página o sección.
nav: Bloque de navegación.
article: Contenido independiente y autocontenido (ej. un post de blog).
section: Sección temática genérica.
footer: Pie de página.
main: Contenido principal y único de la página.
aside: Contenido relacionado pero secundario (barra lateral).

Beneficios
Accesibilidad: Ayuda a lectores de pantalla a interpretar la estructura para usuarios con discapacidad. SEO (Optimización para Buscadores): Mejora la comprensión de Google sobre tu contenido, ayudando a posicionarte mejor.
Mantenibilidad: Hace el código más legible y fácil de mantener para otros desarrolladores.
Futuro: Prepara tu sitio para estándares web futuros, ya que estas etiquetas son más resilientes que las no semántica
